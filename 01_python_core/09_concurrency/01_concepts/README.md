# 并发与并行 (Concurrency vs Parallelism)

本章节介绍并发编程的核心理论基础。

## 核心知识点

1. **并发 (Concurrency)**：在同一时间间隔内，多个任务交替执行。宏观上同时，微观上轮流（单核 CPU 场景）。
2. **并行 (Parallelism)**：在同一时刻，多个任务真正地同时执行。需要多核 CPU 支持。
3. **阻塞与非阻塞**：程序在等待某个操作（如 IO）时的状态。
4. **同步与异步**：任务执行顺序的调度方式。
