# 类型检查器概览

类型检查器是静态分析工具，它们在不实际运行 Python 代码的情况下，通过扫描源代码来发现类型不匹配和其他潜在的逻辑错误。

## 目录

- [01_what_is_checker.py - 什么是类型检查器](01_what_is_checker.py)：介绍静态类型检查的基本概念及其与运行时的关系。
- [02_comparison.py - 主流检查器对比](02_comparison.py)：以代码注释的形式对比 mypy 和 pyright 的差异。
- [03_selection_guide.py - 选择建议](03_selection_guide.py)：提供针对不同项目背景的选择指南。

## 为什么使用类型检查器？

1. **提前发现 Bug**：在代码部署到生产环境之前，发现参数类型错误或 `None` 引用问题。
2. **辅助重构**：当你修改一个函数签名时，检查器能瞬间指出所有受影响的调用点。
3. **更好的 IDE 支持**：检查器为编辑器提供更精确的自动补全和内联提示。
4. **代码即文档**：强迫开发者编写更清晰、自描述的代码。

## 核心工具

- **mypy**：官方且成熟的检查器，拥有庞大的插件生态系统。
- **pyright**：由微软开发，性能极快，是 VS Code (Pylance) 的核心引擎。
