# Pydantic 综合案例

本章节通过一个模拟的电商订单系统，展示了如何在复杂的现实业务逻辑中综合运用 Pydantic 的各种特性。

## 目录

- [01_ecommerce_order_system.py - 电商订单系统](01_ecommerce_order_system.py)：实现一个包含商品项、收货地址和订单主体的综合系统。

## 涉及特性

该案例演示了以下 Pydantic 功能的协同工作：
1. **嵌套模型**：订单包含地址和商品列表。
2. **枚举支持**：使用 `Enum` 定义订单状态。
3. **字段约束**：价格必须 > 0，电话号码符合正则。
4. **计算字段**：自动计算订单项小计和订单总额。
5. **自定义验证**：确保订单至少包含一个商品。
6. **智能转换**：处理日期和数值。

## 学习建议

- 仔细阅读代码中关于 `@computed_field` 和 `@field_validator` 的使用，理解它们如何保证业务规则。
- 尝试传入不合法的地址信息或空商品列表，观察 Pydantic 如何精确地定位并报告错误。
