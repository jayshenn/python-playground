# 为什么需要类型系统 (Why Type Systems)

本章节介绍 Python 类型提示（Type Hints）的背景、价值及其在现代开发中的地位。

## 核心知识点

1. **类型系统的价值**：提高代码可读性、减少低级错误、增强 IDE 补全能力。
2. **动态 vs 静态**：理解 Python 作为动态语言，如何通过 PEP 484 引入可选的静态检查。
3. **最佳实践**：在大型项目、库开发和 CI/CD 流程中强制使用类型检查。

## 文件列表

- `01_type_system_value.py`: 演示类型提示如何让代码意图更清晰。
- `02_dynamic_vs_static.py`: 对比动态类型带来的灵活性与潜在风险。
- `03_best_practices.py`: 总结现代 Python 项目中应用类型系统的建议。
